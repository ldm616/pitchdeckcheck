# 03 — Paid Fix Rules

The paid report is the improvement layer. It should reveal exact fixes, provide actionable advice, and use investor-backed examples.

## Paid report purpose

The paid report answers:

- What exactly should I change?
- What proof points should I add?
- How should I rewrite this slide?
- Which investor-backed examples are relevant?
- What would make this deck more fundable?
- How do I raise the grade?

## Paid report includes

Deck-level:
- Overall fix plan
- Top 5 highest-impact improvements
- Recommended slide order
- Missing slides to add
- Slides to cut, merge, or simplify
- Narrative improvements
- Investor objection handling
- Highest-priority evidence gaps

Slide-level:
- Exact weak investor questions
- Specific additions or enhancements
- Specific proof points to add
- Suggested headline/subhead/bullets
- Relevant investor-backed examples
- Before/after explanation where useful
- Visual/product-shot suggestions where useful
- What not to say

## Paid report should still avoid

- revealing hidden scoring weights
- dumping the full rubric
- exposing the full precedent library
- showing internal prompt architecture
- listing every possible example
- using the same examples repeatedly when they are not relevant

## Context-specific fix logic examples

Consumer health / self-guided physical therapy app:
- customer trust
- health-adjacent credibility
- behavior-change product experience
- consumer app retention
- clinical or rehab domain insight
- acquisition credibility in health, fitness, or wellness

B2B SaaS:
- buyer pain
- workflow urgency
- budget owner
- sales motion
- retention and expansion potential
- integration depth
- vertical or horizontal wedge

Marketplace:
- supply acquisition
- demand acquisition
- liquidity
- take rate
- repeat behavior
- chicken-and-egg strategy
- network effects

AI infrastructure / developer tools:
- technical credibility
- developer adoption
- workflow integration
- switching costs
- model or data advantage
- open-source risk
- enterprise expansion path

Cybersecurity:
- urgency of threat
- buyer credibility
- trust
- technical differentiation
- compliance/regulatory pull
- sales cycle
- incumbent displacement
